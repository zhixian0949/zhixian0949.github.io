Java基础-注解机制

注解是Java5引入的新特性。对注释我们是比较熟悉的，我们通过注释可以获取一些类、方法和参数的信息，但是这些信息仅仅是提供给人阅读的，虚拟机并不会获取其中的信息。那么注解就是面向虚拟机的注释。举个例子，程序看到@Test注解，就知道这是个测试用例看到@Data，就会添加get和set方法，等等等

注解基础

  • 作用：

    1. 生成Javadoc文档
    2. 编译检查，比如springboot里的实体验证
    3. 编译时动态处理，lombok插件里的几个注解
    4. 运行时动态处理，spring里的bean的生成

  • 元注解

    @Target:描述注解使用范围,比如接口、方法、参数等

    @Retention 保留时间

    @Documented:生成javadoc文档时用不用保留注解信息

    @Inherited子类用不用继承父类的注解

  • 内置注解

    @Override、@Deprecated和@SuppressWarnings作用就不说了
